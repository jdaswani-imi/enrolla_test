## Module Overview

[[07_Operations-M16_Task_Management|M16]] is the task management and workflow module for Enrolla. It provides a unified inbox for all operational tasks across the platform — whether created manually by staff, auto-generated by the [[07_Operations-M13_Automation_Communications|M13]] automation engine, or produced by the approval gateway in [[01_Foundation-PL02_RBAC|PL-02]]. Every task created anywhere in Enrolla lands in [[07_Operations-M16_Task_Management|M16]].

# 01.1 Task Fields

Every task in [[07_Operations-M16_Task_Management|M16]] has the following fields.

| **Field** | **Specification** |
|---|---|
| Title | Required. Short descriptive name. Shown on all task cards in all views. Maximum 120 characters. |
| Description | Optional. Full detail of what needs to be done. Supports @mentions to tag staff. No character limit. |
| Task type | Required. One of: Admin, Academic, HR, Finance, Student Follow-up, Cover, Complaint, Personal. Drives filter behaviour and visibility rules. |

> **Cover task creation:** Cover tasks are generated automatically by the system when a teacher is marked absent and sessions require cover ([[04_Academic-M05_Timetabling_Scheduling|M05]]). Admin and above can also create Cover tasks manually — for example to chase a pending cover assignment. Teachers and TAs cannot create Cover tasks.
| Priority | Required. High, Medium, or Low. Drives the due-date traffic-light colour and sort order within grouped views. |
| Due date | Required. DD/MM/YYYY. Tasks created with a past due date are blocked — the system flags this and the creator must set a future date. Traffic light: red = overdue, amber = due within 24 hours, green = more than 24 hours away. |
| Assignee | Optional. One or more staff members. If no assignee is set, the task is personal to the creator. If an assignee is set, the task is visible to the assignee, creator, their HOD, Admin Head, and Super Admin. |
| Department scope | Optional. Set when a task applies to an entire department or the whole office. Department-scoped tasks are visible to all staff in that department. Office-scoped tasks are visible to all staff. |
| Linked record | Optional. Any record in Enrolla: student profile, guardian profile, invoice, concern, complaint, staff profile, session. Tap the linked record chip to open the record in a side panel without leaving [[07_Operations-M16_Task_Management|M16]]. **Note:** Personal tasks cannot be linked to external records (student profiles, guardian profiles, invoices, etc.). The Linked Record field is not available when Task type is set to Personal. If a task requires a linked record, a different task type must be selected. |
| Sub-tasks | Optional. A checklist of smaller steps within the task. Each sub-task has a title and a checkbox. Progress is shown as X/Y complete on the task card in all views. |
| Recurrence | Optional. Personal recurring reminders: any staff member. Department or office-wide recurring tasks: Admin Head and above only. Each recurrence generates a new task instance automatically. |
| Created by | System-set. The staff member who created the task, or 'System' for automation-generated tasks. Immutable. |
| Created on | System-set. DD/MM/YYYY HH:MM. Immutable. |
| Completed on | System-set. DD/MM/YYYY HH:MM when the task is marked complete. Immutable once set. |

# 01.2 Task Types and Visibility

The task type determines default visibility and filter behaviour. Complaint tasks have special privacy rules — they are not visible to general staff even if searched. @Mentions override default visibility for all types except Complaint.

| **Task Type** | **Default Visibility** |
|---|---|
| Admin | Assignee + creator + HOD of assignee's department + Admin Head + Super Admin |
| Academic | Assignee + creator + HOD + Academic Head + Admin Head + Super Admin |
| HR | Assignee + creator + HR/Finance role + Admin Head + Super Admin only. HR tasks are not visible to HOD or other staff. |
| Finance | Assignee + creator + HR/Finance role + Admin Head + Super Admin only |
| Student Follow-up | Assignee + creator + HOD + Admin Head + Super Admin |
| Cover | Assignee + creator + HOD + Admin Head + Super Admin. Cover tasks auto-notify all department staff as a heads-up. |
| Complaint | Assignee + complaint logger + HOD + Admin Head + Super Admin only. Not searchable or visible to any other staff member regardless of department. |
| Personal | Creator only. Not visible to any other staff member. Does not appear in department views or Admin Head oversight. Not searchable by others. |

# 01.3 Priority and Due Date Traffic Light

Every task displays a priority badge (High, Medium, Low) and a due date traffic light colour. The traffic light reflects the relationship between the current date and the task's due date.

| **Traffic Light** | **Condition** | **Visual Treatment** |
|---|---|---|
| Red | Task due date has passed — overdue | Red left border on task card in List view. Red due date text. Red indicator dot on Kanban card. |
| Amber | Task due within the next 24 hours | Amber left border on task card. Amber due date text. |
| Green | Task due more than 24 hours from now | No border highlight. Standard due date text. |
| Grey | Task completed | Strikethrough on title. Greyed card. No traffic light. |

# 01.4 Views

[[07_Operations-M16_Task_Management|M16]] provides four views accessible from the top navigation bar of the module. All views share the same filter bar and show the same underlying task data. The active view persists per user between sessions.

## 01.4.1 List View

The default view. Tasks are displayed as rows in a scrollable list, grouped by status (Overdue, Today, Upcoming, Completed). Each task row shows: checkbox, title, priority badge, task type badge, due date with traffic light, linked record chip, and assignee avatar. Clicking a task row opens the task detail panel on the right without navigating away from the list.

| **Feature** | **Detail** |
|---|---|
| Default grouping | Overdue → Today → Upcoming → Completed. Collapsed by default when empty. |
| Smart grouping | Staff can switch grouping to: Assignee (Admin Head view), Department, Task Type, or Priority. Grouping persists per user. |
| Sort within group | By due date (default), by priority, by creation date, or alphabetically by title. |
| Inline completion | Tick the checkbox on a task row to mark it complete without opening the detail panel. The row moves to Completed instantly. |
| Quick-add | A single-line task creation input at the top of the list. Title, assignee, due date, priority. Tap to expand for full fields. Saves with Enter key. |
| Bulk select | Checkbox column header selects all visible tasks. Bulk actions: mark complete, reassign, change due date, change priority, delete. |

## 01.4.2 Kanban View

Tasks are displayed as cards organised into three columns: To Do, In Progress, Done. Cards are draggable between columns — dragging a card to Done marks it complete.

| **Feature** | **Detail** |
|---|---|
| Columns | To Do, In Progress, Done. Column counts shown in the column header badge. |
| Card drag-and-drop | Drag a card between columns to update its status. Dropping on Done marks the task complete and logs the completion timestamp. |
| Card content | Title (truncated at 2 lines), priority badge, assignee avatar, due date with traffic light colour, linked record chip, sub-task progress indicator (X/Y). |
| Column filter | Each column can be filtered independently by task type, priority, or assignee using the column header dropdown. |
| Overdue indicator | Red border on overdue cards in all columns including In Progress. Overdue tasks cannot be hidden in Kanban view. |
| Collapse columns | Each column can be collapsed to a narrow strip to focus on a subset of statuses. |

## 01.4.3 Calendar View

Tasks are plotted on a calendar by their due date. Day, week, and month toggles available. Tasks with no due date do not appear in Calendar view.

| **Feature** | **Detail** |
|---|---|
| Toggles | Day, week, and month views. Week view is the most useful for daily planning. |
| Colour coding | Task cards are coloured by priority: red = High, amber = Medium, green = Low. Overdue tasks show with a red indicator regardless of priority colour. |
| Multi-day tasks | Tasks that span multiple days show as a bar across the calendar. |

## 01.4.4 Snooze

Snooze is a task-level action, not a field. Any staff member can snooze a task they can view to temporarily hide it until a future date. Snoozed tasks disappear from all views until the snooze date arrives. A snooze reason is required. Snooze count is visible on the task card on return. After 3 snoozes on a Medium, High, or Urgent priority task, Admin Head is notified.

# 01.5 Task Detail Panel

The task detail panel opens in a right-side panel when a task is clicked. The panel displays all task fields and provides the action bar for completing, editing, reassigning, and commenting on the task.

| **Element** | **Detail** |
|---|---|
| Header | Task title (editable), priority badge, task type badge, status selector (To Do / In Progress / Done) |
| Fields section | All task fields — editable inline where the staff member has permission |
| Sub-task checklist | Interactive checklist. Each item ticked or unticked. Progress bar at top. |
| Linked record context strip | Shown when a student is linked (see 01.6). |
| Comment thread | Persistent threaded discussion. @Mentions supported. File attachments (PDF, image, Word) supported per comment. Comments cannot be deleted — soft editing only within 5 minutes. |
| Activity log | Chronological list of all changes to the task: field edits, assignments, completions, snoozes. Each entry shows actor name, change made, and timestamp. |
| Action bar | Mark Complete, Snooze, Reassign, Duplicate, Delete. |

# 01.6 Linked Record Context Strip

When a task is linked to a student profile, the task detail panel displays a context strip below the task header. This keeps task management fast — the handler has full context in one screen.

| **Context Strip Element** | **Detail** |
|---|---|
| Student name and year group | Full name and current year group label |
| Churn risk score | Current overall churn score with colour badge (green < 40, amber 40–69, red 70+). High Churn Flag badge shown if active. |
| Open concerns | Count of currently open concerns for this student. Clicking the count opens the concern list in a side panel. |
| Outstanding invoices | Count of unpaid or overdue invoices. Clicking opens the invoice list in a side panel. |
| Last contact date | Date of the most recent logged contact with the guardian. Shown in DD/MM/YYYY format. |
| Quick link to full profile | One-tap link to open the full [[03_Student-M17_Student_Profile|M17]] student profile in a side panel. |

# 01.7 Recurring Tasks

Recurring tasks auto-generate new task instances on a configured schedule. When the current instance is completed, the next instance is automatically created with the same fields and a new due date.

| **Recurrence Setting** | **Detail** |
|---|---|
| Who can create | Personal recurring tasks: any staff member. Department or office-wide recurring tasks: Admin Head and Super Admin only. |
| Recurrence patterns | Daily, Weekly (specific day of week), Fortnightly (every 2 weeks), Monthly (specific date of month or specific day-of-month, e.g. 'first Monday of the month'), Custom (every N days / every N weeks / every N months — N is a configurable integer). |
| Instance generation | The next instance is created automatically when the current instance is marked complete. The new due date is calculated from the completion date + recurrence interval — not from the original due date. This prevents drift accumulation. |
| Edit recurrence | Editing the recurrence pattern affects future instances only. The current open instance is unchanged. |
| End condition | No end date (runs indefinitely), End after N instances (N configurable), End by a specific date. |
| Pause recurrence | The recurrence can be paused without deleting it. No new instances are generated while paused. Resuming generates the next instance from the resume date. |
| Delete recurrence | Deleting the recurrence cancels all future instances. The current open instance and all completed instances remain in the audit trail. |

# 01.8 Task Templates

Task templates are pre-configured task structures — title, type, sub-task checklist, and default assignee role — that can be applied with a single click. Templates save time for recurring operational workflows.

## 01.8.1 Template Types

Personal templates are created by any staff member and visible only to themselves. Org-wide templates are created by Admin Head or Super Admin and visible to all staff.

## 01.8.2 IMI Default Org-Wide Templates

| **Template Name** | **Sub-tasks Included** |
|---|---|
| Staff off-boarding checklist | Revoke system access / Collect lanyard and access card / Reassign open sessions / Handover document signed / Final payroll confirmed / Departure announcement (if applicable) |
| New student onboarding | Profile mandatory fields complete / Invoice issued / Sessions scheduled / Folder and materials issued / Parent communication sent / Progress tracker created |
| Concern follow-up sequence | Initial parent contact logged / L1 review completed with teacher / Intervention plan documented / Parent update sent / Follow-up scheduled |
| Invoice overdue chase | First contact attempt logged / Second contact attempt (3 days) / Admin Head notified (7 days) / Payment plan offered (if applicable) |
| Re-enrolment outreach | Contact parent 3 weeks before term / Follow up 1 week before term / Follow up 4 days before term / Confirm or close enrolment |
| Cover assignment | Identify available cover teacher / Confirm cover assignment / Notify students (copy-paste message generated) / Cover session marked in timetable |
| Event preparation checklist | Room confirmed / Materials prepared / Attendance list printed / Parent notifications sent / Post-event attendance marked |

# 01.9 Bulk Actions

Bulk actions allow Admin Head and Super Admin to perform operations across multiple tasks simultaneously. Bulk actions are available in List view via the checkbox column.

| **Bulk Action** | **Detail** |
|---|---|
| Mark complete | Marks all selected tasks as complete. Completion timestamp logged for each. Tasks move to Completed group. |
| Reassign | Opens an assignee picker. Selected tasks are reassigned to the chosen staff member. Original assignee is notified of the reassignment. |
| Change due date | Opens a date picker. Selected tasks all receive the new due date. Traffic lights update immediately. |
| Change priority | Opens a priority selector. Selected tasks all receive the new priority. |
| Change type | Opens a type selector. Visibility rules may change for affected tasks. |
| Delete | Permanently deletes selected tasks. Requires confirmation. Auto-generated tasks (created by the [[07_Operations-M13_Automation_Communications|M13]] automation engine or the [[01_Foundation-PL02_RBAC|PL-02]] approval gateway) cannot be deleted by the task creator or assignee. They can only be moved to Completed status. Admin Head and Super Admin can archive auto-generated tasks but cannot delete them. The audit log permanently retains all auto-generated task records. Complaint tasks cannot be bulk-deleted — they must be individually deleted by Admin Head or Super Admin. |
| Snooze | Opens snooze options. Selected tasks all snoozed to the same date and reason. |

Bulk action confirmation is required when the selection exceeds 10 tasks.

# 01.10 Notifications

[[07_Operations-M16_Task_Management|M16]] generates the following in-app notifications. All [[07_Operations-M16_Task_Management|M16]] notifications are in-app only in v1.

| **Trigger** | **Recipient** | **Notes** |
|---|---|---|
| Task assigned to staff member | Assignee | Includes task title, due date, priority, and link to [[07_Operations-M16_Task_Management|M16]] task detail. |
| Task assignee is on leave at assignment time | Task creator | Soft alert only — 'This person is currently on leave and may not see this task'. |
| Task due in 24 hours — not started | Assignee | Amber traffic light fires simultaneously. |
| Task overdue — past due date | Assignee + Admin Head | Fires at end of due date. Red traffic light fires simultaneously. |
| Task snoozed 3 times without completion | Admin Head | Snooze count shown on task card. |
| High priority task unactioned for 24 hours | Admin Head | High priority tasks only. |
| Task completed | Task creator | Confirmation. Does not fire if creator and assignee are the same person. |
| @mention in task description or comment | @mentioned staff member | In-app notification with the comment preview and a link to the task. |
| Task linked student has since withdrawn | Task owner | Prompt to close, reassign, or keep. |

# 01.11 Role-Based Access Control

| **Role** | **Create Tasks** | **View Tasks** | **Manage Others' Tasks** |
|---|---|---|---|
| Super Admin | All types including Complaint | All tasks — full visibility across all staff and departments | Full — can edit, reassign, delete any task |
| Admin Head | All types including Complaint | All tasks except Personal tasks of other staff | Full — can edit, reassign, delete any non-personal task |
| Academic Head | All except HR and Finance | All Academic, Admin, Cover, Student Follow-up tasks in all departments | Can reassign and edit Academic tasks |
| HOD | All except HR, Finance, and Complaint | All tasks in own department except HR, Finance, and Personal of others | Can reassign tasks within own department |
| Teacher | Admin, Academic, Cover, Student Follow-up, Personal | Own tasks + tasks assigned to them + department tasks they are tagged in | Own tasks only |
| TA | Academic, Cover, Personal | Own tasks + tasks assigned to them | Own tasks only |
| Admin | Admin, Finance, Student Follow-up, Personal | Own tasks + tasks assigned to them + Admin Head-scoped department tasks | Own tasks only |
| HR/Finance | HR, Finance, Personal | HR and Finance tasks + own personal tasks | Own tasks only |
